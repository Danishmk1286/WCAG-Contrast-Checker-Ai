# Contributing to Color Contrast Checker Demo

Thank you for your interest in contributing! This is an educational repository focused on demonstrating WCAG 2.1 color contrast checking functionality.

## What Can I Contribute?

We welcome contributions in the following areas:

- **Documentation**: Improve README, add examples, clarify explanations
- **Examples**: Add new use cases, integration examples
- **Bug Fixes**: Fix issues in demo code
- **Code Quality**: Improve code readability, add comments
- **Accessibility**: Enhance component accessibility
- **Tests**: Add unit tests for core algorithms
- **UI Improvements**: Enhance demo components (keep them simple)

## Getting Started

1. **Fork the repository**
2. **Clone your fork**
   ```bash
   git clone https://github.com/yourusername/color-contrast-checker-demo.git
   cd color-contrast-checker-demo
   ```
3. **Install dependencies**
   ```bash
   npm install
   ```
4. **Create a branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Development Guidelines

### Code Style

- Use TypeScript for type safety
- Follow existing code style
- Add comments for complex logic
- Keep components simple and educational

### Commit Messages

Use clear, descriptive commit messages:

```
feat: Add example for batch color checking
fix: Correct contrast ratio calculation edge case
docs: Update README with new examples
```

### Pull Request Process

1. **Update documentation** if needed
2. **Test your changes** locally
3. **Ensure code follows guidelines**
4. **Submit pull request** with clear description

### Testing

Before submitting:
- Run `npm run build` to ensure code compiles
- Test in browser to verify functionality
- Check that examples work correctly

## Code of Conduct

- Be respectful and inclusive
- Focus on educational value
- Help others learn
- Keep discussions constructive

## Questions?

If you have questions about contributing:
- Open an issue for discussion
- Check existing documentation
- Review example code
